## 2024-05-24 - [Auth Bypass] Undefined Environment Variable in Template Literals
**Vulnerability:** When checking `Authorization` headers or URL query parameters against environment variables like `process.env.CRON_SECRET` using string interpolation (e.g., `` `Bearer ${process.env.CRON_SECRET}` ``), if the environment variable is not configured, it evaluates to the string `"undefined"`. An attacker can simply pass `"Bearer undefined"` or `?key=undefined` to bypass authentication checks.
**Learning:** This is a common JavaScript/TypeScript pitfall where undefined values in template literals do not fail or throw an error, but instead cast to the string `"undefined"`. Relying purely on string comparison without verifying the existence of the secret leads to a critical auth bypass, essentially "failing open" rather than "failing closed".
**Prevention:** Always explicitly check that the security-critical environment variables are truthy (e.g., `!process.env.CRON_SECRET`) before performing authentication validation. Alternatively, use a safe getter function that throws an error if a critical secret is missing.
